Front End Developement

I have been building web applications using the core front end technologies (HTML/CSS/JavaScript) and also explored and experimented with other frameworks such as the Handlebars templating markup and the Meteor framework.

I've added HTML5 functionality to applications (Nyise web App) using LocalStorage, Geolocation and WebSockets which has been interesting and have had the opportunity to experiment with Google GeoLocations API (FrostBytes web-app) including features such as distance calculations and map markers.

I have also began working with SASS at a very basic level declaring variables and mixins, and working with nested statements.

I really enjoy working with JavaScript and with each project am becoming more confident with the language. The applications I am working on have allowed me to understand more intermediate concepts such as closures, promises, different data structures and new ES6 collections.

Although I'm comfortable with HTML/CSS but I know that I need to focus more on my core CSS as my projects in the past have relied on a frameworks such as Bootstrap. I have been taking online courses provided by Lynda and Treehouse and handcoded a responsive grid to gain better understanding of how modern frameworks work

Back End Developement

After seeing great things happen using the Meteor framework while building FrostBytes, I realized I didn't actually understand how each individual component of the application was working...I was just gluing bits together, This encouraged me to begin working with Node/Express, MongoDB and Handlebars separately to understand what each component did within the application, and how it did it.

I began by serving static pages from my own node server and then working with express to serve dynamic content pulled from a noSQL databases, rendered client side with help of Handlebars templating markup.

I have implemented persistence with a noSQL database in both Node and Java and wrote simple create and read statements . I have begun studying SQL database fundamentals/terminology but I have yet actually implement anything.

I also studied part of the CISCO CCNA syllabus and experimented with GNS3 network simulator building simple LAN and WAN networks for a better understanding of how networks are built, configured and operate.

I understand that I still have quite limited knowledge of the back-end , but I know that I'm on the right road with getting the basics down and building more complex applications as the time goes on.

Where possible, we would like to see mostly CSS 1 & 2 usage. CSS 3 & 4 should be limited to widely-supported elements only, or strong fallbacks should be provided. The CSS Zen Garden is about functional, practical CSS and not the latest bleeding-edge tricks viewable by 2% of the browsing public. The only real requirement we have is that your CSS validates.

Luckily, designing this way shows how well various browsers have implemented CSS by now. When sticking to the guidelines you should see fairly consistent results across most modern browsers. Due to the sheer number of user agents on the web these days — especially when you factor in mobile — pixel-perfect layouts may not be possible across every platform. That’s okay, but do test in as many as you can. Your design should work in at least IE9+ and the latest Chrome, Firefox, iOS and Android browsers (run by over 90% of the population).
